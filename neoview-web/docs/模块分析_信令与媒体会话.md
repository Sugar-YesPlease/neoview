# 信令与媒体会话 模块分析

## 1. 功能概述 (Functional Overview)
该模块负责 WebSocket 信令连接与 mediasoup 会话协商，完成 join、transport 建立、producer/consumer 生命周期管理，并将房间事件统一回调到 `App.vue`。

## 2. 页面跳转流程 (Page Transition Flow)
```mermaid
stateDiagram-v2
    state "未连接" as Idle
    state "信令连接中" as WsConnecting
    state "协商中" as Negotiating
    state "会话已建立" as Connected
    state "已关闭" as Closed

    Idle --> WsConnecting : "调用join(roomId)"
    WsConnecting --> Negotiating : "protooOpen"
    Negotiating --> Connected : "join成功"
    Connected --> Closed : "leave或socket关闭"
```

## 3. 接口清单 (API List)
| Interface Description | URI | Method | Parameter Description | Code Reference |
| :--- | :--- | :--- | :--- | :--- |
| 获取路由能力 | `protooRequest:getRouterRtpCapabilities` | WS Request | 无 | `src/services/mediasoupSession.js` |
| 创建WebRTC传输 | `protooRequest:createWebRtcTransport` | WS Request | `producing, consuming` | `src/services/mediasoupSession.js` |
| 连接传输 | `protooRequest:connectWebRtcTransport` | WS Request | `transportId, dtlsParameters` | `src/services/mediasoupSession.js` |
| 入会 | `protooRequest:join` | WS Request | `displayName, device, rtpCapabilities` | `src/services/mediasoupSession.js` |
| 发布轨道 | `protooRequest:produce` | WS Request | `transportId, kind, rtpParameters, appData` | `src/services/mediasoupSession.js` |

## 4. 业务逻辑时序图 (All Business Logic)
### 4.1 建立信令连接
```mermaid
sequenceDiagram
    participant A as "App.vue"
    participant MS as "mediasoupSession.js"
    participant SG as "signaling.js"

    A->>MS: "join(roomId)"
    MS->>SG: "connect()"
    SG-->>MS: "WebSocket open"
    MS->>SG: "connectToRoom(roomId)"
    SG-->>MS: "protooOpen"
```

### 4.2 创建发送与接收Transport
```mermaid
sequenceDiagram
    participant MS as "mediasoupSession.js"
    participant SG as "signaling.js"
    participant SFU as "服务端"

    MS->>SG: "request(createWebRtcTransport, producing=true)"
    SG->>SFU: "protooRequest"
    SFU-->>SG: "transport参数"
    SG-->>MS: "返回sendTransport数据"
    MS->>SG: "request(createWebRtcTransport, consuming=true)"
    SG->>SFU: "protooRequest"
    SFU-->>SG: "transport参数"
    SG-->>MS: "返回recvTransport数据"
```

### 4.3 入会协商
```mermaid
sequenceDiagram
    participant MS as "mediasoupSession.js"
    participant SG as "signaling.js"
    participant SFU as "服务端"

    MS->>SG: "request(getRouterRtpCapabilities)"
    SG->>SFU: "protooRequest"
    SFU-->>SG: "rtpCapabilities"
    SG-->>MS: "返回能力"
    MS->>MS: "device.load(rtpCapabilities)"
    MS->>SG: "request(join,{displayName,device,rtpCapabilities})"
    SG->>SFU: "protooRequest"
    SFU-->>SG: "join成功"
    SG-->>MS: "connected"
```

### 4.4 处理服务端下发新消费者
